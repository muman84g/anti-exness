Purpose: push live bot files quickly while keeping live_config out of GitHub, then tell the user the CentOS-side next steps.

## First-Time Bot Push Rule

For a first-time bot push, include the generated target bot folder as-is.

Exclude only:

- `live_config.py`
- direct live_config derivatives, especially `__pycache__/live_config*.pyc`
- `.env`, credential files, secrets, tokens, and account credentials

This means initial `logs/`, `state/`, `sNN_bot_state.json`, other `__pycache__/*.pyc`, and generated placeholder files can be committed on the first push.

For an existing bot update, stage only changed files unless the user asks to refresh the whole folder.

## Fast Route For bot20

```powershell
$STAMP = Get-Date -Format yyyyMMdd_HHmmss
$CLONE = "C:\botter\_push_anti-exness-$STAMP"
git clone https://github.com/muman84g/anti-exness.git $CLONE
```

Copy bot20 as a folder, then remove only live_config artifacts:

```powershell
$SRC = 'C:\botter\bot'
$DST = $CLONE
Copy-Item -LiteralPath "$SRC\BOT_BACKTEST_MAP_ja.md" -Destination "$DST\BOT_BACKTEST_MAP_ja.md" -Force
Get-ChildItem -LiteralPath $SRC -Filter 'github*push.txt' | Select-Object -First 1 | Copy-Item -Destination $DST -Force

Remove-Item -LiteralPath "$DST\bot20\live_config.py" -Force -ErrorAction SilentlyContinue
Remove-Item -LiteralPath "$DST\bot20\__pycache__\live_config.cpython-312.pyc" -Force -ErrorAction SilentlyContinue
```

```powershell

git diff --cached --name-only | Select-String -Pattern '(^|/)live_config\.py$|__pycache__/live_config|\.env'
```

The final command must return no matches.

Commit and push:

