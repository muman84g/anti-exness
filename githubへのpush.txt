GitHubへのpush手順メモ

## 基本方針

- `live_config.py`、`.env`、秘密鍵、トークン、認証情報、本番設定ファイルはgitに含めない。
- `*_bot_state.json` / `s*_bot_state.json` はgitに含めない。必要なstate JSONはCentOS側へ手動で配置する。
- `state/` のような空ディレクトリをgitで作りたい場合は、`state/.gitkeep` などのplaceholderだけをcommitする。
- `logs/`、`__pycache__/`、`.pyc`、実行ログ、取引CSVはgitに含めない。
- push前に `git status --short` と `git diff --cached --name-only` で、意図しないbotや機密ファイルが混ざっていないか確認する。

## 新規bot追加と既存bot更新の違い

- 新規bot追加: 対象botディレクトリ一式、`docker-compose.yml`、`AGENTS.md`、必要な運用メモをstageする。ただし `live_config.py` と実state JSONは含めない。
- 既存bot更新: 対象botディレクトリ丸ごとではなく、変更した実ファイルだけをstageする。`docker-compose.yml` や `AGENTS.md` は変更した場合だけstageする。
- どちらの場合も、stage後に `git diff --cached --name-only` を見て、意図しないbot、削除、機密ファイル、state JSONが混ざっていないことを確認する。

## 汚れた作業ツリーを避けるpush例

既存repoの作業ツリーに大量の未commit変更や削除がある場合は、そこを直接触らず、一時ディレクトリへクリーンcloneして必要ファイルだけ反映する。

```bash
git clone https://github.com/muman84g/anti-exness.git /tmp/anti-exness-push
cd /tmp/anti-exness-push

# 必要ファイルだけコピーしてから確認
git status --short
git add AGENTS.md docker-compose.yml githubへのpush.txt bot18
git status --short
git diff --cached --name-only

# live_config.py や state JSON が含まれていないことを確認してからcommit/push
git commit -m "Add bot18 live bot files"
git push origin main
```

## 通常のpush例

作業ツリーがきれいな場合だけ、通常repo上で以下を行う。

```bash
git status --short
git add AGENTS.md docker-compose.yml githubへのpush.txt bot18
git diff --cached --name-only
git commit -m "Add bot18 live bot files"
git push origin main
```

`git add bot18` を使う場合も、`.gitignore` により `bot18/live_config.py` と `bot18/state/s18_bot_state.json` は通常はstageされない。念のため `git diff --cached --name-only` で必ず確認する。
